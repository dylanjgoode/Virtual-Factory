MQTT In Motion

Devices publish telemetry, alarms, state, and status into the broker. The broker fans out those topics to the dashboard, controller, and any observer while caching retained state for instant onboarding.

Publish

Devices stream telemetry (QoS 0) and alarms (QoS 1) into topic lanes like telemetry/{device}/{sensor} and alarms/{device}/{sensor}.

Retain + LWT

State and status are retained by the broker; LWT flips a status offline when a client disappears uncleanly.

Fan-out

Controller and dashboard subscribe broadly, so the broker can replay retained state and deliver new alarms instantly.

Commands

Dashboard + controller publish commands (QoS 1) to commands/{sender}/{device}, and devices respond on state/status.
